tasks dashboard · design exploration

Eight ways to make the tasks dashboard calmer.

The current dashboard packs eight metric cards, a five-column kanban, an agent allocation panel, a watch list, an autopilot strip, and a message bus into a single screen. These mocks each take a different position on what to keep, what to demote, and what to delete. The first four were the practical exploration; the second four push harder on minimalism and humane interface principles.

01
One number, not eight.
Replace the metric carpet with a single header line. Counts return inline only where they're load-bearing.
02
Color is information.
Accent for active. Warn for blocked. Ok for done. Everything else greyscale. No surface tinting.
03
Don't demand attention; offer it.
The dashboard should reward a glance, not require one. Live elements earn their motion; static elements earn their stillness.
v1 · status-first
List + Rail
open →
A single grouped list — Now / Up Next / Blocked / Done — with a side rail for crew and a recent-events pulse. Click any task to open a detail drawer with subtasks and per-task activity. Includes empty and completed states (cycle them via the switcher in the top right).
Best when users want to scan task state at a glance and triage. Most familiar; least new metaphor. Fewest pixels for the mission itself.
v2 · mission-first
Reading Room
open →
A single 640px column. The mission gets serif display type and the largest pixels. Tasks read as paragraphs with status as marginalia. Crew and recent activity demoted to a quiet footer strip with a glass cradle. No rail, no panels, no chrome.
Best when the mission deserves contemplation, the task list is short-to-medium (≤20), and you want the platform to feel like a writing tool rather than a control room.
v3 · time-first
Timeline
open →
Time across the X axis, agents as horizontal lanes. Past tasks as solid bars, current as glowing pulse, queued as dashed ghosts. A "now" line marches right across the canvas. Queue and blocked tasks land in a card row underneath.
Best when cadence and crew utilization matter more than backlog shape. Makes idle agents and pile-ups visible at a glance — and lets you tell at a glance whether the mission is keeping pace.
v4 · shape-first
Constellation
open →
Tasks float as soft cards in status zones — done dimming away, now glowing in the center, next waiting to the right, blocked amber and isolated. Faint connecting lines hint at dependency. Agents shown as labeled orbs near what they're touching.
Best when you want the mission's shape to become legible — dependency clusters, abandoned branches, central bottlenecks. The most spatial / least scannable; best as an alternate view rather than the default.
pass two · humane + minimal
Calmer takes that follow Raskin-style principles — modeless, monotonous, attention-respecting.
v5 · prose-first
Narrative Stream
open →
The dashboard is a curated log written by the agents themselves. Tasks emerge as inline pills inside chronological prose. A "currently" card at the foot of the stream is the only live element — everything above is past tense. Quiet stretches collapse into a single fold-line.
Best when the work is mostly autonomous and you check in periodically. Eliminates the modal split between "chat" and "dashboard" — they become one continuous reading surface.
v6 · attention-first
Spotlight
open →
Just one task. In display serif, centered, with a breathing cursor. A row of progress dots above. Everything else demoted to whisper-text below: who's working alongside, what's queued, who's idle. Use ←/→ to step through.
Best when the mission has a clear critical path. Maximally attention-respecting — answers "what is the next right thing?" in a single glance and refuses to show you anything else.
v7 · static-first
Daily Brief
open →
Calm typewritten memo on cream paper. No live elements, no animation. Mission, status table, crew, highlights since you last looked, what needs you, what's queued, sign-off. Refresh when you want to. Open the live view at any time.
Best when you've stepped away and want to scan the state without entering a live monitoring mode. Could ship as the "tab back in" view, an emailed daily, or a calmer alternative for async-minded users.
v8 · keyboard-first
Keyboard
open →
A polished TUI in the browser. Every task has a short slug (s3-stream, azure) and every action has a single-key shortcut shown in the bottom command bar. j/k nav, claim, block, xcomplete, / search. The prompt accepts both natural language and structured commands like :claim azure forge.
Best when the user is power-user-shaped. Modeless, monotonous, predictable — Raskin's humane principles literally applied. The only variant where speed and minimalism reinforce each other instead of trading off.
