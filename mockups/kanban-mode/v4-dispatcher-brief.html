§ 2Policies
stale after
A claim is stale if the agent makes no tool call for 12 min.
retry policy
If a card fails, retry up to 2× with the same agent before reassigning.
circuit breaker
Pause autopilot if more than 3 failures occur in 10 min.
max in flight
No more than 1 card per agent at a time.
spend cap
Pause & ask the operator if mission spend exceeds $5.00.

§ 4What you should never do
  • ·Auto-dispatch a "cutover / risky" card. These always wait for the operator.
  • ·Reclaim a card before it's gone stale. Even slow agents need a chance to think.
  • ·Dispatch a card whose dependencies are unmet. Wait for the upstream card to finish first.
  • ·Send the same card to the same agent twice in a row after a failure. That's what the retry-then-reassign rule is for.
