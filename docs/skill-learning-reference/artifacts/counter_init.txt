## design notes

Defends against AttributeError on first nudge-check by setting safe defaults BEFORE the try/except config read — if `_agent_cfg.get("memory")` raises or returns malformed data, the counters and intervals still exist with sensible defaults (10 turns / 10 iters), so `_turns_since_memory += 1` and the modulo check never crash mid-conversation. Note the asymmetry a porter must preserve: `_turns_since_memory` and `_iters_since_skill` are runtime counters initialized to 0 (incremented per turn/iter), while `_memory_nudge_interval` and `_skill_nudge_interval` are config-driven thresholds defaulting to 10 (compared against the counters). Both interval reads use `int(...)` to coerce YAML strings/floats and are wrapped in bare try/except so a bad config value falls back to 10 rather than raising. The `skip_memory` gate only skips the config read — counters are set unconditionally above the gate, which is critical because skill nudging runs even when memory is skipped (the two systems share counter scaffolding but are independently gated). Caveat for porters: `nudge_interval` (memory) lives under `agent.memory.*` while `creation_nudge_interval` (skills) lives under `agent.skills.*` — the YAML key names differ and are NOT interchangeable.</design_notes>
