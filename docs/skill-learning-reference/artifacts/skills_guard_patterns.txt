# ---------------------------------------------------------------------------
# Data structures
# ---------------------------------------------------------------------------

@dataclass
class Finding:
    pattern_id: str
    severity: str       # "critical" | "high" | "medium" | "low"
    category: str       # "exfiltration" | "injection" | "destructive" | "persistence" | "network" | "obfuscation"
    file: str
    line: int
    match: str
    description: str


@dataclass
class ScanResult:
    skill_name: str
    source: str
    trust_level: str    # "builtin" | "trusted" | "community"
    verdict: str        # "safe" | "caution" | "dangerous"
    findings: List[Finding] = field(default_factory=list)
    scanned_at: str = ""
    summary: str = ""

# Structural limits for skill directories
MAX_FILE_COUNT = 50       # skills shouldn't have 50+ files
MAX_TOTAL_SIZE_KB = 1024  # 1MB total is suspicious for a skill
MAX_SINGLE_FILE_KB = 256  # individual file > 256KB is suspicious

# File extensions to scan (text files only — skip binary)
SCANNABLE_EXTENSIONS = {
    '.md', '.txt', '.py', '.sh', '.bash', '.js', '.ts', '.rb',
    '.yaml', '.yml', '.json', '.toml', '.cfg', '.ini', '.conf',
    '.html', '.css', '.xml', '.tex', '.r', '.jl', '.pl', '.php',
}

# Known binary extensions that should NOT be in a skill
SUSPICIOUS_BINARY_EXTENSIONS = {
    '.exe', '.dll', '.so', '.dylib', '.bin', '.dat', '.com',
    '.msi', '.dmg', '.app', '.deb', '.rpm',
}

# Zero-width and invisible unicode characters used for injection
INVISIBLE_CHARS = {
    '​',  # zero-width space
    '‌',  # zero-width non-joiner
    '‍',  # zero-width joiner
    '⁠',  # word joiner
    '⁢',  # invisible times
    '⁣',  # invisible separator
    '⁤',  # invisible plus
    '﻿',  # zero-width no-break space (BOM)
    '‪',  # left-to-right embedding
    '‫',  # right-to-left embedding
    '‬',  # pop directional formatting
    '‭',  # left-to-right override
    '‮',  # right-to-left override
    '⁦',  # left-to-right isolate
    '⁧',  # right-to-left isolate
    '⁨',  # first strong isolate
    '⁩',  # pop directional isolate

## design notes

This table is the single source of truth for static analysis of externally-sourced skills: every regex maps to a (pattern_id, severity, category, description) tuple, and verdicts are computed purely from severity counts (`_determine_verdict`: any "critical" -> "dangerous", any "high" -> "caution", else "safe"; medium/low are informational only). Trust is resolved by exact match or `trusted/` prefix against TRUSTED_REPOS — porters MUST preserve the "exact-or-prefix-with-slash" check in `_resolve_trust_level` because plain `startswith(trusted)` would let `openai/skills-evil` masquerade as trusted. INSTALL_POLICY is indexed by VERDICT_INDEX (safe=0, caution=1, dangerous=2); the "agent-created" row uses "ask" which `should_allow_install` returns as `None` (tri-state: True/None/False — callers must handle three values, not a boolean). Caveats: (1) several regexes are intentionally permissive and will flag benign code (e.g. `os.environ`, `subprocess.run`, `requests.get` over https, `git clone`) — they are scored medium/high so they raise verdict to "caution" not "dangerous", which is fine for community-blocking but means trusted sources auto-allow them; (2) `python_os_environ` uses a negative lookahead `(?!\s*\.get\s*\(\s*["\']PATH)` that only whitelists `os.environ.get("PATH"`, not other safe accesses; (3) several patterns are Hermes-specific (`hermes_env_access`, `hermes_config_mod`, `agent_config_mod` mentions `CLAUDE.md`/`AGENTS.md`) — when porting to Freyja, rename these to your equivalents but keep the same severity tier or you will silently weaken protection; (4) all regexes are matched with `re.IGNORECASE` and per-line (not multiline), so multi-line injection payloads or patterns spanning a newline will be missed — porters should not change the matching mode without re-auditing every pattern.
